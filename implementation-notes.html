Digia Engage — Analytics Event Matrix Implementation Notes

Branch: aditya/analytics-changes (digia_engage + digia_engage_ios). Backend: feature/analytics. Goal: fulfil the full engage-event-matrix across Flutter, React Native, Android, iOS. Started 2026-06-17.

Locked decisions (from grilling)

  • DECISION RN seam: JS emits raw lifecycle only; native (Android/iOS) owns 100% of Digia first-party analytics. RN never builds or sends a matrix event.
  • DECISION CEP plugins keep their JS p.track() forwarding (separate system, untouched).
  • DECISION Flutter scope: analytics only for types Flutter renders — inline (carousel/stories) + nudge (bottom_sheet/dialog). guide/survey are UnsupportedCampaignConfig in Flutter and are skipped.
  • DECISION Fidelity: full matrix incl. first-class trigger_type/trigger_event plumbing through the trigger path + plugin API on every platform.
  • DECISION Backend is the fixed contract; only concrete gaps get fixed (see below). E2E via npm run local (:3000).
  • DECISION Event model: ONE AnalyticsService per platform. Coarse DigiaExperienceEvent (CEP + existing tests) stays untouched. Add ONE capture(eventName, ctx, properties) entry; renderers assemble matrix properties at the call site. No new event-type hierarchy.
  • DECISION Build order: Flutter → Android → iOS → RN. TDD throughout.

Concrete backend gap found

  • FOR YOU enrich.ts derives the item_index/item_total columns only from step_index/question_index; it ignores item_index/item_total, which inline events use directly. Agreed fix: num(props.step_index ?? props.question_index ?? props.item_index) (and the same for _total). item_id/story_id/frame_id correctly stay in residual properties JSON. You will commit backend changes.

Design decisions & deviations (running log)

Flutter

  • Baseline: all 13 tests in analytics_service_test.dart pass before any change.
  • DONE Added DigiaAnalyticsService.capture(eventName, {campaignKey, campaignType, campaignId, properties, flush}) — the single matrix entry point. Refactored the shared enqueue/payload-builder so captureExperienceEvent (CEP/coarse path) is behaviourally identical. 14/14 tests green (13 original + 1 new tracer-bullet test).
  • DECISION Matrix properties are passed as a Map<String,dynamic> assembled by the caller using exact matrix snake_case keys (e.g. display_style, item_index, time_to_action_ms). The service merges static context (sdk_version, device_*) + identity (anonymous_id/session_id/user_id) + top-level campaign fields. No per-event Dart type — matches "one service, no new event hierarchy".
  • DECISION flush:true on capture() is how terminal events (Experience Dismissed/Completed) force an immediate flush, mirroring the existing flushOnCapture behaviour that the CEP sink applies for coarse events.
  • DONE Added EngageEventEmitter.analytics(eventName, payload, {properties, flush}) — render surfaces' entry point for Digia-only matrix events. The campaign-resolution + capture() call is injected as an AnalyticsDispatch at the composition root (DigiaInstance), keeping the emitter decoupled/testable. 2 new emitter tests; 16/16 green.
  • DECISION Matrix (Step/granular) events route Digia-only via emitter.analytics(); coarse CEP lifecycle still goes via toCep/toAll. So a nudge "viewed" will fire toCep(ExperienceImpressed) (CEP) and analytics('Digia Experience Viewed', …rich props) (Digia) — CEP stays coarse, Digia gets full fidelity.
  • DONE Pure matrix property builders (lib/api/internal/event/engage_matrix.dart) assemble each event's properties map with exact snake_case keys, unit-tested in isolation (engage_matrix_test.dart) — the error-prone mapping logic is covered without heavy widget tests. Renderer wiring then calls these at the emitter.analytics() site. (Renderer glue itself is verified by the builder tests + medihub E2E rather than widget tests — a deliberate TDD tradeoff for the UI plumbing.)
  • DECISION Coarse→CEP / rich→Digia split. Each user moment now fires the coarse DigiaExperienceEvent to CEP only (toCep) and the rich matrix event to Digia only (analytics()). Nudge present/dismiss/click were changed from toAll/toDigia → toCep + analytics() so Digia is not double-counted.
  • DONE Nudge fully instrumented: Digia Experience Viewed (display_style bottom_sheet/dialog, screen_name, trigger_type/trigger_event), Clicked (element_id, cta_label, cta_position, action_type/action_url, time_to_action_ms), Dismissed (time_to_dismiss_ms, flush). View clock lives on DigiaInstance (single active nudge).
  • DEVIATION Nudge Clicked now fires for any actionable button (primary or any button carrying actions), not just primary as before — required to populate cta_position=secondary per the matrix. Consequence: CEP also now sees a coarse Clicked for secondary CTAs.
  • DECISION element_id synthesised as cta_primary/cta_secondary — the Flutter NudgeButton model has no server-assigned button id. Deterministic and mirrors cta_position.
  • DONE Inline carousel: Experience Viewed (display_style=carousel, item_total, screen/trigger context) fired once per campaign via an extended digiaImpressionOnce(eventName, properties) that keeps the per-campaign dedup but emits the rich event instead of coarse ExperienceImpressed. Step Viewed for slide 0 at first render + on every onPageChanged; Step Clicked on slide tap (action_type=deeplink/action_url when the slide has a link).
  • DONE Inline stories: card tap → Experience Clicked (item_index = opened frame); per-frame Step Viewed in _startItem; CTA tap → Step Clicked; early close (swipe-down/back) → Step Dismissed; playing through the last frame → Experience Completed (time_to_complete_ms, flush). A _terminated guard prevents double terminal events. Story widgets resolve their payload from the slot key (the viewer is a pushed route), same pattern as the nudge renderer reaching DigiaInstance.
  • DECISION Inline granular events are Digia-only (analytics()), since CEP's coarse impression/dismiss for inline are already fired instantly at route time (syncTemplate semantics). Story open/step/complete therefore do not also hit CEP.
  • DECISION Trigger attribution is first-class. Added optional triggerType (event|screen_view|manual|app_open) + triggerEvent to CEPTriggerPayload (backward-compatible — both nullable). Threaded into every "Viewed" event. screen_name now stored on DigiaInstance.setCurrentScreen regardless of whether a CEP plugin is registered.
  • OPEN Nudge dismiss_reason (backdrop|close_button|back_button|secondary_cta|auto_timeout) is not emitted on Flutter — nudge_presenter.dart completes its dismiss Future without surfacing the cause. Only time_to_dismiss_ms is sent. Threading a reason through the presentation strategies is deferred; confirm whether this fidelity is required for Flutter.
  • OPEN item_id / frame_id absent on Flutter inline: neither CarouselItem nor StoryItemConfig carries a server id, so item_id is omitted (only item_index/item_total are sent). If the backend/dashboard needs a stable per-item id on Flutter, the config models must parse one.
  • Flutter test suite: 25/25 green (13 original analytics + 1 capture tracer + 3 emitter + 8 matrix-builder).

Android

  • Baseline: the test module did not compile on checkout — DigiaInstanceTest.kt referenced the old InAppPayload-at-the-boundary contract and a FakePlugin.notifyEvent(event, InAppPayload) signature that no longer existed after the CEPTriggerPayload rename. This blocked all Android tests. Confirmed via a clean compile that this breakage predates my changes.
  • DEVIATION Rewrote DigiaInstanceTest.kt faithfully to the current contracts: trigger delivery via CEPTriggerPayload(cepCampaignId, campaignKey) at the delegate boundary, FakePlugin.notifyEvent(event, CEPTriggerPayload), events captured as Pair<DigiaExperienceEvent, CEPTriggerPayload>, and assertions against the real nudgeOverlay/slotPayloads channels. Restores the "existing tests pass 100%" invariant.
  • DECISION Coarse→CEP / rich→Digia split via one seam: DisplayCoordinator.emitMatrix(coarse, eventName, payload, properties, flush) — fires the coarse DigiaExperienceEvent to pluginRegistry.notifyEvent (CEP) only when non-null, and always the named matrix event to analyticsService.capture(eventName, …) (Digia). Mirrors the Flutter emitter. Renderers/orchestrators call DigiaInstance methods that delegate to this single seam.
  • DONE Pure matrix property builders (internal/analytics/EngageMatrix.kt) mirror engage_matrix.dart exactly (nudgeViewed/nudgeClicked/nudgeDismissed/containerViewed/step/completed/question + actionTypeToken/actionUrlOf). Unit-tested in isolation (EngageMatrixTest.kt, 8 tests) — UI glue verified by builder tests + integration tests + device E2E, same TDD tradeoff as Flutter.
  • DONE Nudge: reportNudgeImpression (Impressed→CEP + rich Viewed with display_style/screen/trigger), emitNudgeClick(label,isPrimary,actions) (Clicked(cta_$position)→CEP + rich Clicked), markNudgeDismissed(reason?) (Dismissed→CEP + rich Dismissed, flush). View clock on DigiaInstance. NudgeNodeRenderer now fires the click for any button that is primary or carries actions (matches the Flutter deviation, populates cta_secondary).
  • DONE Inline carousel (DigiaComposableApi.InlineCarouselView): LaunchedEffect(payload.id) fires reportInlineViewed (Impressed→CEP + rich Viewed) + reportInlineStep(0); a snapshotFlow { currentPage % pageCount }.distinctUntilChanged().drop(1) fires Step Viewed on slide change; slide .clickable fires Step Clicked (deeplink action_type/url) and opens the deeplink.
  • DONE Inline story: card tap (DigiaInlineStory) → emitStoryOpened = Experience Clicked (element_id=story_card, item_index=opened frame); the fullscreen overlay (DigiaStoryOverlay) fires per-frame Step Viewed via LaunchedEffect(currentStoryIndex), Experience Completed (time_to_complete_ms, flush) in onCompleted, and Step Dismissed via a DisposableEffect.onDispose guarded by a terminated flag so completion / CTA exits don't also count as a dismiss.
  • DECISION Inline & story granular events are Digia-only (coarse=null); only the container Viewed sends the coarse Impressed to CEP — matching the Flutter inline decision (CEP impression/dismiss for inline are coarse-only).
  • DONE Survey enriched onto the matrix: reportSurveyStarted now emits Impressed→CEP + rich Experience Viewed (display_style=standard, item_total=question-node count); reportSurveyQuestionViewed → Digia-only Question Viewed; reportSurveyAnswered keeps the one-shot internal Experience Clicked engagement signal and now also fires per-question Digia-only Question Answered (item_index/item_total/question_id/question_type + answer); markSurveyDismissed → Dismissed→CEP + rich Experience Dismissed (flush). SurveyRenderer fires Question Viewed from a LaunchedEffect(node.id) for non-content blocks.
  • DECISION question_type = the block type enum lowercased (e.g. single_select, nps); item_index/item_total computed over answerable nodes only (content blocks — welcome / text-media / result-page — excluded). question_id = survey node id.
  • OPEN Survey Question Skipped has no Android trigger: the Android survey renderer has no skip CTA, so reportSurveyQuestionSkipped exists for parity/RN/iOS but is currently unwired on Android. Confirm whether Android surveys should expose a skip affordance.
  • DONE Guides (tooltip/spotlight) — previously fired nothing. Now: reportGuideViewed (Impressed→CEP + rich Experience Viewed, display_style from step 0, item_total=step count) once per guide when the anchor resolves; reportGuideStepViewed per step; emitGuideStepClick on CTA advance (Digia-only Step Clicked); completeGuide on advancing past the last step (Dismissed→CEP + rich Experience Completed, flush); dismissGuide on early close (Digia-only Step Dismissed for the abandoned step + Dismissed→CEP + rich Experience Dismissed, flush). Wired in GuideRenderer via LaunchedEffects keyed on campaignKey/stepIndex.
  • DEVIATION Old dismissGuide built an InAppPayload with empty content (losing campaign_id). Replaced with a proper guidePayload(campaign) carrying campaign_id/key/type=guide/display_style, so guide analytics are correctly attributed.
  • DONE Added integration tests to DigiaInstanceTest for the survey (start→CEP Impressed, question/answer stay off CEP; dismiss→CEP Dismissed) and guide (view→CEP Impressed, completion→CEP Dismissed, clicks stay off CEP; early dismiss clears state + CEP Dismissed) routing invariants.
  • Android test suite: 75/75 green (was 63 broken → 71 after the nudge/inline/story slice → 75 after survey + guide tests). Property correctness covered by EngageMatrixTest (8); routing/lifecycle by DigiaInstanceTest (16).
